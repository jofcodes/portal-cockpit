# Cockpit — Portal app

A real, installed Android app for **Meta Portal** that displays the Productivity
Cockpit. It's a thin **WebView wrapper** (`com.josephine.cockpit`) — one
full-screen Activity that renders the self-contained dashboard generated by
`cockpit/build_cockpit.py`, with a JS bridge (`window.Cockpit.exit()`,
`refresh()`). Cloned from the beehive monitor's `portal_app/`.

> You can also run the Cockpit **without building this app** by pointing the
> Portal's browser at the Mac server (`http://<lan-ip>:8899/`). See
> [`../COCKPIT.md`](../COCKPIT.md). The native app is the polished,
> no-browser-chrome option.

## Build & install (run in your own Terminal)

The APK build uses Gradle's daemon, which needs a local socket the assistant's
sandbox blocks — so build it yourself:

```bash
cd cockpit_app
./deploy.sh        # generate dashboard → build APK → install + launch on Portal
```

`deploy.sh` auto-detects Android Studio's bundled JDK and your Android SDK
(`~/Library/Android/sdk`). Prereqs: Portal connected via USB-C with **ADB
Enabled** (Settings → Debug), `adb` installed, and Android Studio (for the JDK).

## Refresh after new data (no rebuild)

```bash
cd cockpit_app
./refresh.sh           # regenerate dashboard + push to Portal
./refresh.sh --jobs    # also run the data jobs (brief, inbox, wrap) first
```

## Files
```
cockpit_app/
├── deploy.sh / refresh.sh                       # build+install / push
├── settings.gradle, build.gradle, gradle.properties
└── app/
    ├── build.gradle                             # plain Java, no AndroidX; minSdk 24, targetSdk 34
    └── src/main/
        ├── AndroidManifest.xml                  # launcher activity; NO INTERNET permission
        ├── java/com/josephine/cockpit/MainActivity.java   # WebView + Cockpit JS bridge
        ├── assets/dashboard/index.html          # generated (gitignored)
        └── res/                                 # icon (gauge), strings, fullscreen theme
```

## Policy
- The app uses **no network** (no INTERNET permission) — all content is local.
- Outbound actions are staged as drafts upstream (in the jobs), never auto-sent.
- Summaries/drafts use **local Llama** via Ollama bound to `127.0.0.1`.
